Tabela Pagamentos:
id ,Vencimento ,Valor,Status, id_fornecedor,id_conta, id_funcionario, id_categoria,id_forma_pagamento,CHAVE, data_pagamento,observacoes, anexos

Campo	Tipo	Descrição
id	INT (PK)	Identificador único do pagamento.
data_vencimento	DATE	Data de vencimento do pagamento.
valor	DECIMAL(10, 2)	Valor total do pagamento.
status	ENUM('Pendente', 'Pago', 'Atrasado')	Status do pagamento (pendente, pago ou atrasado).
id_fornecedor	INT (FK)	Referência ao fornecedor, se for pagamento a fornecedor.
id_funcionario	INT (FK)	Referência ao funcionário, se for pagamento de salário.
id_conta	INT (FK)	Referência à conta a ser paga, como água, energia, etc.
id_categoria	INT (FK)	Categoria do pagamento (ex: fornecedor, imposto, salário, etc.).
id_forma_pagamento	INT (FK)	Forma de pagamento (boleto, cartão de crédito, transferência, etc.).
data_pagamento	DATE	Data em que o pagamento foi efetuado.
observacoes	TEXT	Campo para observações extras sobre o pagamento.

 Relacionamentos
Agora, vamos definir os relacionamentos entre as tabelas:

Pagamentos e Fornecedores: Um pagamento pode estar associado a um fornecedor (relacionamento de 1 para N, onde um fornecedor pode ter vários pagamentos).

Pagamentos e Funcionários: Um pagamento pode ser destinado a um funcionário (relacionamento de 1 para N, onde um funcionário pode ter vários pagamentos de salários).

Pagamentos e Contas: Um pagamento pode estar relacionado a uma conta de consumo (água, luz, etc.) (relacionamento de 1 para N).

Pagamentos e Formas de Pagamento: Cada pagamento pode ser feito de uma forma específica (boleto, transferência, etc.) (relacionamento de N para 1).

Pagamentos e Categorias de Pagamento: Cada pagamento pertence a uma categoria (relacionamento de N para 1, com categorias como Fornecedor, Imposto, Salário, etc.).
